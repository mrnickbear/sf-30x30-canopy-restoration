Executive Summary

Project Mission

This project delivers a production-grade LiDAR processing pipeline, built with the lidR package for R, that extracts fine-grained canopy structure metrics from the 2023 USGS San Francisco 3D point cloud. The pipeline feeds an interactive web dashboard giving field managers an evidence-based tool for targeted biodiversity interventions.

The primary study area is Laguna Honda Hospital (LHH) and its surrounding open spaces — a representative example of the urban green infrastructure at the heart of San Francisco's 30×30 strategy. By combining individual tree segmentation, snag classification, and understory thicket detection, the outputs directly inform where limited restoration resources will have the greatest ecological impact.

The analysis targets three key canopy threats: invasive English ivy, Himalayan blackberry thickets, and senescent / snag eucalyptus. Each represents a distinct structural signature detectable at the point-cloud level, enabling automated classification at scale without costly manual field surveys for initial prioritization.

Data & Methodology

Source Data: 2023 USGS LiDAR of San Francisco

The foundational dataset is the 2023 USGS 3DEP LiDAR acquisition covering the city and county of San Francisco. Tiles are sourced from the USGS National Map Downloader and stored locally at data/raw_point_clouds/.

Technical justification for fine-scale structural analysis: The lidRbook states that accurate tree-species identification typically requires a point density of at least 30 pts/m². The 2023 USGS project report confirms this dataset achieves a maximum density of 200 pts/m² — nearly 7× the minimum threshold. This ultra-high density makes fine-scale structural analysis (including identification of understory thickets beneath a closed canopy) highly viable without additional data collection.

Processing Pipeline

Raw LiDAR tiles are processed through a reproducible four-stage R pipeline. Each stage produces audited intermediate outputs so any step can be re-run independently.

01
Load & Clip Catalog the raw point cloud tiles and clip to the LHH analysis area boundary (data/vector/updatedAA.kml).
02
Normalize, Segment & Classify Height-normalize returns, detect tree tops using the Li et al. (2012) point-cloud algorithm1, segment individual crowns, and classify snag status. Saves segmented LAS to disk.
03
Crown Metrics & Visualization Compute per-tree crown metrics (height, area, snag class) and export data/vector/crowns.geojson for the web dashboard.
04
Web Point Cloud Preparation Export one buffered, uncompressed .las per tall tree into data/web_point_clouds/ for the interactive 3D viewer.
